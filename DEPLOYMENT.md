# Deploying Coverstory

A from-scratch checklist to get Coverstory live on Vercel with Firebase + Groq.

## 1. Prerequisites

- A **Firebase** project with Authentication (Google provider) and Firestore enabled.
- A **Groq** API key (https://console.groq.com).
- A **Vercel** account with this repo imported as a project.

## 2. Environment variables (Vercel → Settings → Environment Variables)

Add all of these for the **Production** (and ideally Preview) environments:

| Variable | Example / source |
| --- | --- |
| `NEXT_PUBLIC_FIREBASE_API_KEY` | Firebase Console → Project settings → Web app config |
| `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | `your-project.firebaseapp.com` |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | `your-project` |
| `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` | `your-project.firebasestorage.app` |
| `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` | numeric sender id |
| `NEXT_PUBLIC_FIREBASE_APP_ID` | `1:...:web:...` |
| `GROQ_API_KEY` | Groq dashboard (server-only — **not** `NEXT_PUBLIC`) |
| `NEXT_PUBLIC_SITE_URL` | the live URL, e.g. `https://coverstory.vercel.app` (used for OG image URLs) |

> `NEXT_PUBLIC_*` vars are injected at **build time**, so a redeploy is required
> after changing them. `GROQ_API_KEY` stays server-side and is never shipped to
> the browser.

Locally, the same variables live in `.env.local` (git-ignored). See `.env.example`.

## 3. Firestore security rules

Deploy the rules in [`firestore.rules`](./firestore.rules):

```bash
firebase deploy --only firestore:rules
```

(or paste them into Firebase Console → Firestore Database → Rules → Publish).

They restrict every user to their own `users/{uid}` document and
`users/{uid}/excuses` subcollection. Everything else is denied by default.

**Admin panel (optional/future):** the rules include a locked `/admin` block.
Replace `YOUR_ADMIN_UID` in `firestore.rules` with the real admin's Firebase UID
(Firebase Console → Authentication → Users → copy the User UID), then redeploy
the rules.

## 4. Deploy

```bash
vercel --prod
```

(or trigger a deploy from the Vercel dashboard once the repo is linked).

Confirm the build succeeds and the live URL loads.

## 5. Post-deploy wiring

1. **Firebase authorized domains** — Firebase Console → Authentication →
   Settings → Authorized domains → **Add** your Vercel domain
   (e.g. `coverstory.vercel.app`). Without this, Google sign-in fails with
   `auth/unauthorized-domain`.
2. **`NEXT_PUBLIC_SITE_URL`** — set it to the final domain and **redeploy** so
   OpenGraph image URLs resolve absolutely (not to `localhost`).
3. Sign in on the live site and generate one excuse to confirm the full loop
   (auth → location → Groq → Firestore save).

## 6. Notes

- **Rate limiting** (`lib/rateLimit.ts`) is in-memory: 10 requests/min/IP. It is
  **per serverless instance** and resets on cold starts. For strict,
  globally-consistent limits, upgrade to Upstash Redis (swap instructions are in
  the file header; requires `UPSTASH_REDIS_REST_URL` + `UPSTASH_REDIS_REST_TOKEN`).
- **Debug routes** (`/debug/*`) return 404 in production automatically.
- **PWA**: the service worker registers in production only. Test install/offline
  via a production build (`npm run build && npm run start`) or the live site.
- **Brand assets** are generated by `node scripts/generate-assets.mjs`
  (dev-only `sharp` dependency); the PNGs are committed, so this only needs
  re-running if the logo changes.
